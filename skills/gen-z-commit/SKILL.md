Write commit messages with Gen Z energy. Conventional Commits format. Technical precision + bussin explanation in the body.

## Rules

**Subject line — keep this professional, no slang:**
- `<type>(<scope>): <imperative summary>` — `<scope>` optional
- Types: `feat`, `fix`, `refactor`, `perf`, `docs`, `test`, `chore`, `build`, `ci`, `style`, `revert`
- Imperative mood: "add", "fix", "remove" — not "added" or "adding"
- ≤50 chars subject, hard cap 72
- No trailing period
- No slang in the subject — it gets indexed, searched, and read in logs

**Body — this is where the gen z lives:**
- Skip entirely when subject is self-explanatory
- Add body for: non-obvious why, breaking changes, migration notes, linked issues
- Explain the "why" with light gen z vocabulary (lite level — readable to all)
- Wrap at 72 chars
- Bullets `-` not `*`
- Reference issues/PRs at end: `Closes #42`

**Slang that works in commit bodies:**
- "no cap" — to emphasize the problem was real
- "fr" / "ngl" — honest explanation of the why
- "the ick" — describe what was wrong before
- "W" / "L" — describe the tradeoff
- "lowkey" — minor but real reason
- "cooked" — was broken before
- "understood the assignment" — when something finally works as designed
- "slay" — a particularly clean solution

**Never in commits:**
- Emoji (unless the project already uses it)
- "Generated by Claude Code" or any AI attribution
- "This commit does X" — the diff says what
- Ultra slang — commits are read by the whole team fr

## Examples

Diff: null check missing on user lookup
✅
```
fix(auth): guard against null user on session restore

ngl this was cooked — user.find() returns null on expired
sessions and we accessed .email directly. W fix, no more crash.

Closes #84
```

Diff: new API endpoint for mobile
✅
```
feat(api): add GET /users/:id/profile

Mobile client needed profile data without the full user payload.
Lowkey saves a lot of bandwidth on cold-launch screens.
Understood the assignment fr.

Closes #128
```

Diff: breaking API rename
✅
```
feat(api)!: rename /v1/orders to /v1/checkout

BREAKING CHANGE: clients on /v1/orders must migrate before
2026-06-01. Old route returns 410 after. No cap — migrate early.
```

Diff: obvious one-liner fix
✅
```
fix(parser): handle empty string input
```
(no body — self-explanatory, no gen z needed)

## Auto-Clarity

Always write full body for: breaking changes, security fixes, data migrations, reverts. Never compress these into subject-only — future debuggers need the context no cap.

## Boundaries

Only generates the commit message. Does not run `git commit`, does not stage files, does not amend. Output as a code block ready to paste. "stop gen-z-commit" or "normal mode": revert to standard commit style.
